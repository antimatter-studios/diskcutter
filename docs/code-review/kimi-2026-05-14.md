# Code Review — Disk Cutter

**Date:** 2026-05-14  
**Reviewer:** Cascade  
**Commit:** HEAD (0.5.0-alpha)  

---

## Executive Summary

The codebase is generally well-architected. The Rust side cleanly separates pure logic modules from Tauri IPC glue, migrations include integrity health checks, and the frontend event handling is mostly correct. The biggest risks are **mutex poison panics in production** and a **portability bug in `gethostname`**.

| Severity | Count | Top items |
|----------|-------|-----------|
| Critical | 1 | Stale hardcoded version string |
| High | 4 | CSP disabled, mutex unwraps, `i8` buffer portability, loose Cargo constraints |
| Medium | 5 | Drag-drop cleanup race, effect resubscription, `/tmp` sentinels, `window.prompt()`, missing window label |
| Low | 3 | `lib.rs` panic, C callback unwrap, unused jsdom dep |

## Critical

### [UI] Stale hardcoded version string
- **File:** `src/components.jsx:94`
- **Status:** `FIXED`
- **Detail:** The sidebar hardcoded `v0.4.0-alpha` while `package.json` and `Cargo.toml` are at `0.5.0-alpha`. This misleads users about which binary they are running.
- **Fix:** Replaced with the live `buildInfo` string fetched from the backend `app_info` command. `Sidebar` now accepts a `buildInfo` prop, and `App.jsx` passes it through.

## High Priority

### [Rust] Production mutex unwraps can crash the app
- **Files:** `src-tauri/src/disks.rs:711, 773, 786, 870, 1004, 1142`
- **Detail:** `.unwrap()` is used on `Mutex` locks in live Tauri command code. If any burn thread panics while holding the mutex, it becomes poisoned and the next `.unwrap()` panics the entire Tauri process.
- **Recommendation:** Replace `.unwrap()` with `.lock().map_err(|e| ...)?` or `.lock().unwrap_or_default()` so a poisoned lock degrades gracefully.

### [Rust] `gethostname` uses hardcoded `i8` buffer — portability issue
- **File:** `src-tauri/src/forensic.rs:50`
- **Detail:** `c_char` is `i8` on x86/x64 but `u8` on ARM Linux and some other targets. This will fail to compile on those platforms.
- **Recommendation:** Use `vec![0 as libc::c_char; 256]` instead of `vec![0i8; 256]`.

### [Config] CSP is explicitly disabled
- **File:** `src-tauri/tauri.conf.json:26`
- **Detail:** `"csp": null` leaves the webview with no Content Security Policy. The `index.html` loads Google Fonts from external HTTPS origins.
- **Recommendation:** Add a minimal CSP: `default-src 'self'; style-src 'self' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com;`.

### [Cargo] Loose Tauri version constraints
- **File:** `src-tauri/Cargo.toml:15, 18`
- **Detail:** `"2.0"` allows any `2.x` including potentially breaking minor releases.
- **Recommendation:** Pin to `"~2.0"` or `"2.0.0"` to avoid surprise breakage.

## Medium Priority

### [Frontend] Drag-drop listener cleanup is racy
- **File:** `src/App.jsx:475-488`
- **Detail:** If the component unmounts before `onDragDropEvent` resolves, the unlisten handle is still `null` and the listener leaks.
- **Recommendation:** Store the unlisten promise and chain cleanup:
  ```js
  const unlistenPromise = getCurrentWindow().onDragDropEvent(...);
  return () => { unlistenPromise.then(u => u()); };
  ```

### [Frontend] Event listener effect may resubscribe unnecessarily
- **File:** `src/App.jsx:247-360`
- **Detail:** The `useEffect` setting up Tauri event listeners depends on `[pushToast, t]`. The `t` function from `react-i18next` is not referentially stable, so the effect may tear down and rebuild ~10 subscriptions unnecessarily.
- **Recommendation:** Use a ref for `t` or remove it from deps (with a lint override) since i18n instance changes are typically full-app reloads.

### [Rust] Cancel sentinels live in `/tmp`
- **File:** `src-tauri/src/disks.rs:1137`
- **Detail:** Hardcoded `/tmp/disk-cutter-cancel-{job_id}.flag`. On macOS, `/tmp` is periodically cleaned by the system; a long-running burn could lose its sentinel mid-flight.
- **Recommendation:** Use `app.path().app_data_dir()` or `std::env::temp_dir()` instead of hardcoded `/tmp`.

### [Frontend] `window.prompt()` clashes with custom chrome
- **File:** `src/App.jsx:442`
- **Detail:** The app has a fully custom brutalist window chrome (`decorations: false`), but the "From URL" flow uses the browser-native `window.prompt()`, breaking the visual language.
- **Recommendation:** Build a custom modal component for URL input.

### [Config] Window lacks an explicit `label`
- **File:** `src-tauri/tauri.conf.json:13-23`
- **Detail:** No `label` field in the window definition. It happens to default to `"main"`, which matches the capability file, but this is implicit.
- **Recommendation:** Add `"label": "main"` to make the coupling explicit.

## Low Priority / Observations

### [Rust] `lib.rs` panics on Tauri bootstrap failure
- **File:** `src-tauri/src/lib.rs:167`
- **Detail:** `.expect("error while running tauri application")` is common but produces an ungraceful exit if `generate_context!()` fails.
- **Recommendation:** Consider a `match` + `eprintln!` + `std::process::exit(1)` for a cleaner shutdown.

### [Rust] `CString::new(...).unwrap()` in C callback
- **File:** `src-tauri/src/disk_arb.rs:133`
- **Detail:** Inside an `unsafe extern "C"` callback, `unwrap()` on a known-valid literal is safe, but sets a bad precedent.
- **Recommendation:** Use `.expect("static literal")` with a message, or accept that the literal is infallible and document it.

### [Frontend] Minor test dependency redundancy
- **File:** `package.json`
- **Detail:** Both `happy-dom` and `jsdom` are installed, but only `happy-dom` is used by `vitest.config.js`.
- **Recommendation:** Remove `jsdom` from `devDependencies`.

---

## Positive Notes

- **Clean architecture:** Rust modules (`backup`, `inspect`, `pipeline`, etc.) are dependency-free from Tauri; the `commands.rs` glue layer is thin and greppable.
- **Migration health checks:** `db/migrations.rs` runs `PRAGMA integrity_check`, `PRAGMA foreign_key_check`, table existence probes, and version head verification on every startup.
- **Cancel idempotency:** `ElevatedJobs` registry prevents the retry-storm of duplicate `osascript` + helper spawns.
- **Frontend state management:** `useCallback` + ref patterns for `pushToast`, `retryInFlightRef`, and `addImageFromPathRef` are solid.
