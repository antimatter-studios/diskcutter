The architecture is solid: pure Rust logic modules separated from Tauri IPC glue, extensible reader/writer plugin system, migration health checks on every startup. The frontend event model is mostly correct. The biggest risks are **mutex poison panics in production** and a **lossy error path in the pipelined writer** that can hang burns silently.

## High Priority

### 1. Mutex `.unwrap()` in production code can crash the process
- **Files:** `src-tauri/src/disks.rs:711,773,786,870,1004,1142`; `src-tauri/src/writers/pipelined.rs:85,90`
- If any thread panics while holding a `Mutex`, it poisons. The next `.unwrap()` panics the entire Tauri process. In `pipelined.rs`, worker threads call `rx.lock().unwrap()` and `errors.lock().unwrap()` â€” a single poisoned lock kills all workers.
- **Fix:** Replace with `.lock().map_err(|e| format!("{e}"))?` or `.lock().unwrap_or_default()`.

### 2. Pipelined writer: errors collected but never surfaced
- **File:** `src-tauri/src/writers/pipelined.rs:77,85-92`
- Worker threads push I/O errors into a shared `Vec<io::Error>`, but the producer (`Write::write`) never checks it. If a worker hits an error and breaks, the producer keeps sending jobs until the channel fills and blocks forever â€” the burn hangs with no feedback.
- **Fix:** Poll the error vector in the producer loop; abort on first error.

### 3. `DiskClaim::for_dev` can leak the worker thread on timeout
- **File:** `src-tauri/src/disk_arb.rs:261-276`
- If the unmount callback doesn't fire within 30s, the function returns `Err(...)` but the spawned thread is still running its `CFRunLoop`. The `Drop` impl explicitly skips joining ("joining can hang"), so the orphaned thread runs forever. The `CFRunLoopStop` call only happens when `runloop_addr != 0`, but the timeout path may not have received the `Initialized` event yet.
- **Fix:** Call `CFRunLoopStop` unconditionally in the error path, or use a separate channel to signal the worker to exit.

### 4. `gethostname` uses hardcoded `i8` â€” won't compile on ARM Linux
- **File:** `src-tauri/src/forensic.rs:50`
- `c_char` is `u8` on ARM Linux and some other targets. `vec![0i8; 256]` fails to compile there.
- **Fix:** `vec![0 as libc::c_char; 256]`

### 5. CSP is explicitly disabled
- **File:** `src-tauri/tauri.conf.json:26`
- `"csp": null` leaves the webview with no Content Security Policy. Google Fonts are loaded from external origins.
- **Fix:** Add a minimal CSP: `default-src 'self'; style-src 'self' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com;`

### 6. Loose Cargo version constraints
- **File:** `src-tauri/Cargo.toml:15,18`
- `"2.0"` allows any `2.x` including breaking minor releases.
- **Fix:** Pin to `"~2.0"` or `"2.0.0"`.

### 7. `SparseFileWriter` Drop impl can panic on double-finish
- **File:** `src-tauri/src/sparse.rs:90-108`
- Both `finish()` and `Drop` call `self.file.take().expect("file already taken")`. If `finish()` is called explicitly and then `Drop` runs (e.g., via `mem::drop`), the second `.expect()` panics.
- **Fix:** Make `Drop` a no-op if the file was already taken: `if let Some(mut f) = self.file.take() { ... }`

## Medium Priority

### 8. Drag-drop listener cleanup is racy
- **File:** `src/App.jsx:440-456`
- If the component unmounts before `onDragDropEvent` resolves, `cleanup` is still `null` and the listener leaks.
- **Fix:** Store the promise and chain: `return () => { unlistenPromise.then(u => u()); };`

### 9. Event listener effect resubscribes on every `t` change
- **File:** `src/App.jsx:247-345`
- The effect depends on `[pushToast, t]`. `t` from `react-i18next` is not referentially stable, so ~10 Tauri event subscriptions may tear down and rebuild unnecessarily.
- **Fix:** Use a ref for `t` or remove it from deps with a lint override.

### 10. Cancel sentinels hardcoded to `/tmp`
- **File:** `src-tauri/src/disks.rs:1137`
- `/tmp/disk-cutter-cancel-{job_id}.flag`. On macOS, `/tmp` is periodically cleaned; a long-running burn could lose its sentinel mid-flight.
- **Fix:** Use `std::env::temp_dir()` or `app.path().app_data_dir()`.

### 11. `window.prompt()` clashes with custom window chrome
- **File:** `src/App.jsx:409`
- The app uses fully custom brutalist chrome (`decorations: false`), but the "From URL" flow uses browser-native `window.prompt()`, breaking the visual language.
- **Fix:** Build a custom modal component for URL input.

### 12. Window lacks explicit `label`
- **File:** `src-tauri/tauri.conf.json:13-23`
- No `label` field. It defaults to `"main"` which matches the capability file, but this is implicit coupling.
- **Fix:** Add `"label": "main"`.

### 13. `addImageFromPath` closure captures stale `nextCopies`
- **File:** `src/App.jsx:347-375`
- The callback depends on `[t, pushToast]` but uses `nextCopies` from the outer scope. If the user changes the copies stepper and then drag-drops a file, the old count is used.
- **Fix:** Add `nextCopies` to the dependency array.

### 14. `validate_image_contents` command defined but never registered
- **File:** `src-tauri/src/validate.rs:219-237`
- The `#[tauri::command]` is defined but not included in `lib.rs`'s `invoke_handler`. It's unreachable from the frontend.
- **Fix:** Add it to the handler list or remove the dead command.

### 15. Cancel watcher thread in `helper.rs` never joined
- **File:** `src-tauri/src/helper.rs:173`
- A thread is spawned to poll for the cancel sentinel file. If the burn completes without cancellation, the thread keeps polling until process exit. Harmless but wasteful.
- **Fix:** Use a `oneshot` channel or `AtomicBool` to signal completion and let the watcher exit.

### 16. Two diverging `pick_device_io` functions
- **Files:** `src-tauri/src/disks.rs:1322` and `src-tauri/src/helper.rs:361`
- Both functions select a `DeviceIo` implementation but have different signatures and defaults. The `disks.rs` version (in-process burns) doesn't support the pipelined writer or configurable worker counts; the `helper.rs` version (elevated burns) does. This is a latent config inconsistency.
- **Fix:** Unify into a single function in a shared module.

## Low Priority

### 17. `lib.rs` panics on Tauri bootstrap failure
- **File:** `src-tauri/src/lib.rs:167`
- `.expect("error while running tauri application")` â€” a `match` + `eprintln!` + `exit(1)` would be cleaner.

### 18. `CString::new(...).unwrap()` in C callback
- **File:** `src-tauri/src/disk_arb.rs:133`
- Inside an `unsafe extern "C"` callback. The literal is infallible but `.unwrap()` sets a bad precedent. Use `.expect("static literal")`.

### 19. `jsdom` unused dev dependency
- **File:** `package.json`
- Both `happy-dom` and `jsdom` are installed; only `happy-dom` is used by `vitest.config.js`. Remove `jsdom`.

### 20. `unmount_macos` is dead code
- **File:** `src-tauri/src/writers/raw.rs:86-100`
- Marked `#[allow(dead_code)]`, never called. DA-based claim in `disk_arb.rs` handles unmounting now. Remove it.

### 21. `verify_image` is a no-op stub
- **File:** `src-tauri/src/disks.rs:1217-1221`
- Comment says "kept for compatibility." Either implement or remove.

- **Frontend patterns:** `useCallback` + ref patterns for `pushToast`, `retryInFlightRef`, and `addImageFromPathRef` are correct.
- **Snapshot format:** Simple magic + length-prefixed JSON header + raw bytes with SHA-256 integrity â€” easy to inspect and forward-compatible.
- **Test coverage:** Unit tests for pipeline, snapshot round-trips, validation, CLI parsing, URL sanitization, and integration tests for burn+verify.
