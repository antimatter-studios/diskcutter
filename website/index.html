What it does

Built to flash
without lying.

Most flashers stop the moment the bytes leave the buffer. Disk Cutter reads them back, hashes every sector, and shows you what didn't match.

01

Parallel queue

Drop multiple images, pick a different USB or SD card for each, run them in parallel. The sidebar shows live queue / done / failed counts. The danger-arm gate is one-shot per click — no accidental retries.

02

Read-back verification

Every burn is followed by a second pass that re-reads the device and re-hashes it against the source. On mismatch you get the offending LBA, byte offset, expected bytes, and what was actually written — up to the first 256 sectors. Selectable hash algorithm (SHA-256 or vendored xxh64).

03

Inspect before commit

Partition table, filesystem labels, boot signature, and the full decoder chain shown for every queued image. Pre-burn content validation rejects archives masquerading as bootable images (e.g. a .tar.gz of photos with a valid gzip header).

04

Compression + containers

.gz / .xz / .bz2 / .zst stream straight through a chained decoder. .qcow2 / .vhd / .vhdx / .vmdk read sparse virtual disks via am-img-*, emitting zeros for unallocated clusters. Magic-byte detection catches renamed files (ubuntu.iso.gz → ubuntu.iso).

05

Cross-platform, one Rust core

macOS (universal arm64 + x86_64, signed + notarized), Linux (.deb / .rpm / .AppImage), Windows (NSIS). Same UI, same Tauri-2 + React shell, same pipeline. Linux disk enumeration walks /sys/block; Windows uses Win32_DiskDrive.

06

Per-row forensic log

Every job streams a structured log into SQLite (burn_logs) that you can tail live in the expanded row. Mismatches are recorded byte-for-byte. One-click tamper-evident JSON + Markdown export for audit trails. Doctor view for environment self-diagnostic.

07

Tweakable pipeline

Writer impl (raw / block / pipelined), chunk size, worker count, queue depth, hash algorithm, skip-verify, debug logging — every runtime knob lives in the Prefs view. Pipelined raw writer keeps the USB driver queue full for ~5× the throughput of a single-threaded loop on the same hardware.

08

Catalog + URL fetch

Built-in distro picker (Ubuntu, Debian, Fedora, Raspberry Pi OS, …) with remote refresh and an offline fallback. Paste any image URL and the app downloads, hashes, and queues it. Pre-burn target snapshot of the first 4 MiB so a mis-targeted disk is recoverable; auto-eject on success.

About

Why?

The pitch

More info about every image — partition layout, filesystem, boot signature, decoder chain — before you commit. More disk types supported, not fewer. Parallel queue, not one-at-a-time. Per-row logs you can tail. Every knob that matters is tweakable from the Prefs view. And it doesn't look like a hospital form.

It also assumes the write will lie to you and re-reads every byte to prove otherwise.

Stack

Rust + Tauri 2 for the shell. React for the UI. Brutalist type — Space Grotesk and JetBrains Mono. The blade icon is hand-drawn SVG. No frameworks, no telemetry, no analytics.

License

MIT. Fork it, ship a competitor, do what you want. If you write a better verifier, please tell us.

Inspired by

Other tools refused to write to disks we wanted to write to, only flashed one card at a time, and never told us what was actually inside the image. Disk Cutter writes to whatever shows up, runs the queue in parallel, and shows you the partition layout, filesystem, and bootability of every image before you commit.
